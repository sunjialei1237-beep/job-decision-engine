# 如何填写 profile.md

`profile.md` 是整个系统的「你」——决策引擎和话术引擎的所有判断都以它为参照。它被 `.gitignore` 忽略，**不会提交**，所以可以放心写真实信息。

## 怎么开始

```bash
cp profile.example.md profile.md
# 然后编辑 profile.md
```

## 每个字段为什么重要

| 字段 | 引擎怎么用它 |
|---|---|
| **基本情况**（身份/求职类型/地域/到岗） | 影响评分里「与候选人阶段匹配度」权重（占 15%），也决定话术语气 |
| **核心项目**（技术栈 + 你负责的模块 + 可量化结果） | 技能匹配度和话术匹配点都从这里挑。**点名具体模块**（如「检索召回」「答案溯源」「多轮澄清」），别写「做过一个 AI 项目」 |
| **技能栈**（按熟练度） | 决定技能匹配 1-5 分；话术里点哪个技能最相关 |
| **目标方向**（A/B/C 级） | 直接影响评分倾向——A 级方向的岗位会加权 |
| **回避项** | 规则预筛会据此砍岗；评分也会压低（如纯电销、无转正） |
| **加分点** | HR 回复概率判断会用到（开源/比赛/客户经验 = 加分） |

## 写得好的判据

- **具体 > 笼统**：「用 RAG 做过检索召回 + 答案溯源，召准率提升 35%」远好于「熟悉 AI 技术」。
- **数字 > 形容词**：有量化的结果（响应延迟、转化率、用户数）让评分有依据。
- **点名模块名**：话术引擎会从你写的项目模块里挑最相关的绑进开场白——你写得越细，话术越像「真做过这事的人」发的。
- **回避项写死**：明确列出不要的方向（电销/打杂/外包/某行业），预筛才能帮你挡掉。

## 一个例子（虚构）

```markdown
## 基本情况
- 身份：大三在读，找日常实习
- 地域：北上深杭均可，远程优先
- 到岗：一周内

## 核心项目
### 企业知识库问答助手
- 定位：内部文档问答，问什么答什么并给出处
- 技术栈：Python / FastAPI / 向量检索 / 大模型 API
- 我做的：文档检索召回 + 答案溯源 + 多轮澄清
- 结果：首响 <2s，召准率提升 35%

## 技能栈
- 强项：Python、Prompt 工程、RAG 检索优化
- 中等：向量数据库、Docker

## 目标方向
- A 级：AI 解决方案 / AI 售前 / AI 产品
- B 级：AI 应用开发实习

## 回避项
- 纯电销、纯文案、无转正、强外包
```

填完就能 `/job-analyze` 粘贴 JD 或 `/job-filter` 跑批量了。
