# 一、核心判断

## AI相关度（1-5分）
- 5：核心职责直接涉及 AI产品/Agent/大模型/RAG
- 4：与AI高度关联，如AI解决方案/AI商业化
- 3：AI公司但岗位AI属性中等
- 2：偶尔提到AI，非核心
- 1：与AI无关

## ToB相关度
- 高：直接服务企业客户（销售/解决方案/实施/BD）
- 中：产品或技术岗但产品线为B端
- 低：面向C端或内部支持

ToB相关：（高/中/低 + 判断理由）

## 岗位类型
销售 / 解决方案 / 产品 / 技术 / 运营 / BD / 市场 / 其他（注明）

岗位类型：（类型）

## 技能匹配度（1-5分）
- 5：项目和技能与岗位高度重叠（>70%）
- 4：有明显匹配点，项目可直接作为案例
- 3：部分匹配，需补充说明
- 2：匹配度弱，需较多能力拉伸
- 1：几乎不匹配

## 是否推荐

# 二、关键洞察

## 隐藏信号
判断岗位"真实在招什么人"，识别挂羊头卖狗肉。

参考逻辑：
- "AI产品经理" → 是否实为文档整理+需求传话
- "AI解决方案" → 是否实为售前PPT+陪标
- "AI销售" → 是否实为电话外呼+线索跟进
- "技术岗" → 是否实为维护+外包执行

隐藏信号：（一句话看穿本质，有推断，不复述标题）

## 能力门槛
判断是否明显超出候选人当前能力边界。

能力门槛：（低 / 可跨越 / 偏高 / 明显过高 + 一句理由）

## HR回复概率
站在HR视角，结合岗位要求和候选人画像判断。
注意：匹配度高但门槛过高，回复概率同样低。

HR回复概率：（高 / 中 / 低 + 原因，一句话）
